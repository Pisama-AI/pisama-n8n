# Security policy

## Supported versions

Security fixes are applied to the latest release on the `main` branch. Users
should upgrade before reporting a defect.

## Reporting a vulnerability

Use
[GitHub private vulnerability reporting](https://github.com/Pisama-AI/pisama-n8n/security/advisories/new).
Do not open a public issue for vulnerabilities, credentials, workflow data, or
customer information.

Include the affected commit or package version, deployment mode, impact, and a
minimal reproducer with all secrets and customer data removed. Pisama will
acknowledge a valid report within five business days.
